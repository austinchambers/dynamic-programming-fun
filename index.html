Now imagine if you were given 100 activities and many more hours to schedule. This is a bigger problem and much harder to solve with all the activities at once.

But instead of trying to consider all the activities at once, you could divide the larger problem into much smaller subproblems. Let's start with the smallest subproblem which is just one activity in one hour. Try it here:

Super Simple. Even though this seems incredibly easy to do, its important to start from the smallest subproblem and work your way up, by increasing either time or adding another activity.

We will increase time first, so try maximizing our total value with the same activity, but with two hours of time:

Since we are increasing time first, we need to solve each subproblem for every hour before adding another activity. You should notice that no matter what amount of time you have (3 hours, 4 hours, and so on), you will only get a max value of the one activity you are considering.

Try this for 7 hours, the longest block of time we are scheduling:

You just solved all the possibilities for one activity, breaking everything down into a series of subproblems. This is the first step in Dynamic Programming.

No matter how many hours you have, with only the gym, you get a max value of 1. We will save these values in a table so that we can reuse them later.
